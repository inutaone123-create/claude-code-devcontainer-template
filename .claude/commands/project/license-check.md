ライセンスヘッダーチェックコマンドです。全ソースファイルにライセンスヘッダーが含まれているか検証します。

以下の手順で実行してください：

## Step 1: 対象ファイルの収集

以下の拡張子のファイルをワークスペース全体から検索してください（`build/`, `target/`, `obj/`, `bin/`, `.git/`, `node_modules/` は除外）：

| 言語 | 拡張子 | ヘッダー開始パターン |
|------|--------|-------------------|
| Python | `.py` | `"""` を含む3行以内 |
| C++ / C | `.cpp`, `.c`, `.h`, `.hpp` | `/*` を含む1行目 |
| C# | `.cs` | `/*` を含む1行目 |
| Rust | `.rs` | `//!` を含む1行目 |
| Octave / MATLAB | `.m` | `%` を含む1行目 |

## Step 2: ヘッダーの確認

各ファイルの先頭10行を読み込み、以下のいずれかのパターンが含まれているかチェックしてください：
- `License` または `license` という文字列
- `Copyright` または `copyright` という文字列
- `SPDX-License-Identifier` という文字列

## Step 3: 結果の表示

以下の形式で結果を表示してください：

```
## ライセンスヘッダーチェック結果

### ✅ ヘッダーあり（N ファイル）
- path/to/file.py
- ...

### ❌ ヘッダーなし（N ファイル）
- path/to/file.cpp  ← 要対応
- ...

### ⏭️ スキップ（テスト・自動生成ファイル）
- ...

合計: X / Y ファイルにヘッダーあり
```

## Step 4: 修正の提案

ヘッダーなしファイルが存在する場合：
1. `CLAUDE.md` のライセンスヘッダーテンプレートを参照
2. 各ファイルの言語に対応したヘッダーを先頭に追加するか確認をユーザーに求める
3. ユーザーが承認した場合のみ、ヘッダーを追加する

`{{PROJECT_TITLE}}` や `{{LICENSE}}` などのプレースホルダーが未置換の場合は、`/project:init` を先に実行するよう案内してください。
