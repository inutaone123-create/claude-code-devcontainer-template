# project:suggest-skill — スキル候補の提案

直近の作業パターンを分析し、`.claude/commands/` へ追加すべきスキル候補を提案します。

## 実行手順

1. **直近の作業を振り返る**
   - `git log --oneline -20` で直近20件のコミットを確認
   - `ls .claude/commands/project/` で既存コマンドを確認
   - `git diff HEAD~5 HEAD --name-only` でよく変更されるファイルを確認

2. **パターンを検出する**
   以下の観点でパターンを探す：
   - 繰り返し実行されているコマンドや操作
   - 毎回同じ手順が必要な検証・確認作業
   - 複数フェーズで共通して現れる処理
   - 手動でやっていて自動化できる作業

3. **スキル候補を提案する**
   以下の形式で提案をまとめ、`docs/skill_candidates.md` に追記する：

   ```markdown
   ## スキル候補: <スキル名>

   **発見日**: <日付>
   **きっかけ**: <どのような作業中に気づいたか>
   **提案コマンド**: `/project:<コマンド名>`
   **処理内容**:
   - ステップ1
   - ステップ2
   **承認状態**: 候補中 / 承認済み / 却下
   ```

4. **ユーザーに確認を求める**
   提案内容を表示し、「承認しますか？」と問いかける。
   承認された場合のみ `.claude/commands/project/<コマンド名>.md` を作成する。

## 注意事項

- 一時的な作業はスキル化しない（再利用性があるものだけ）
- 既存スキルと重複しないか確認する
- スキルが増えすぎると管理しづらくなるため、月に2〜3個が目安
